# Atoms 实施计划

## 1. 计划目标

本计划用于将 Atoms 从当前的截图与产品文档状态，推进到一个可开发、可联调、可迭代的实际项目。

本计划默认技术路线如下：

- 前端：uniapp + Vue 3 + TypeScript
- 后端：Java + Spring Boot
- 数据库：MySQL
- 首发平台：微信小程序
- 架构方式：单体 REST API

## 2. 实施原则

- 优先打通核心链路，不追求功能面一次性做全
- 先做真实可用的数据结构和接口，再补装饰性能力
- 前后端同时围绕统一 API 契约推进
- 先保证登录、习惯、打卡、专注、进度、反思的主流程
- 内容和监督功能先预留，不进入第一阶段完整交付

## 3. 阶段划分

### Phase 0：项目初始化

**目标：**
建立统一开发基础，补齐文档与目录约定。

**交付物：**

- 根目录项目说明
- 技术架构文档
- API 文档
- 前端目录确认
- 后端目录规划

**前端任务：**

- 确认 uniapp 项目结构
- 确认页面入口和基础配置
- 统一页面命名和模块拆分方向

**后端任务：**

- 确认 Spring Boot 单体项目基线
- 确认 MySQL 作为持久化方案
- 确认模块边界与实体边界

**验收标准：**

- 团队对产品范围、技术栈、接口边界没有歧义

### Phase 1：账号与登录

**目标：**
完成微信小程序登录与用户基础身份建立。

**交付物：**

- 微信登录接口
- token 或 session 机制
- 用户基础资料接口

**前端任务：**

- 接入微信登录能力
- 存储登录态
- 拉取当前用户信息

**后端任务：**

- 对接微信 code 登录流程
- 创建或更新用户
- 输出统一鉴权结果

**联调重点：**

- 登录成功后获取 token
- 用户信息接口可稳定返回

**验收标准：**

- 用户首次进入可完成登录
- 登录态可用于后续业务接口访问

### Phase 2：习惯主链路

**目标：**
完成习惯创建、编辑、归档、今日展示与打卡。

**交付物：**

- 习惯创建接口
- 习惯列表与详情接口
- 今日习惯接口
- 打卡接口

**前端任务：**

- 首页今日习惯页面
- 创建习惯页面
- 习惯详情页面
- 打卡交互

**后端任务：**

- 习惯实体与表结构
- 重复规则存储
- 今日习惯计算逻辑
- 打卡记录落库

**联调重点：**

- 习惯创建后可立刻在今日列表中看到
- 同一习惯当天只能产生一条完成型打卡

**验收标准：**

- 用户能从创建习惯一路走到完成打卡

### Phase 3：专注与进度

**目标：**
完成专注计时、记录关联与进度聚合。

**交付物：**

- 专注开始接口
- 专注完成接口
- 今日专注记录接口
- 进度总览接口
- 单习惯进度接口

**前端任务：**

- 专注页
- 结束专注后的记录入口
- 进度页中的概览卡片与历史展示

**后端任务：**

- 专注记录实体
- 专注与习惯的关联逻辑
- streak、完成率、历史热力图数据聚合

**联调重点：**

- 专注完成后可直接关联到打卡记录
- 首页与进度页的统计口径一致

**验收标准：**

- 用户可以完成一次专注并在进度页看到对应变化

### Phase 4：反思系统

**目标：**
补齐“执行后复盘”能力。

**交付物：**

- 创建反思接口
- 查询反思列表接口
- 进度页反思展示

**前端任务：**

- 反思输入页
- 反思列表展示
- 空状态展示

**后端任务：**

- 反思实体与表结构
- 按用户、按习惯维度查询
- 最近与全部视图支持

**联调重点：**

- 习惯反思和身份反思都能正确关联

**验收标准：**

- 用户完成打卡后可提交反思并再次查看

### Phase 5：二期能力

**目标：**
预留但不强制纳入首个可用版本。

**能力范围：**

- Mindset 内容库正式接口
- 内容收藏
- 监督伙伴邀请
- Hall of Fame

**原则：**

- 二期能力不得阻塞一期主流程交付

## 4. 推荐实施顺序

建议按以下顺序推进：

1. 后端先建用户、习惯、打卡三类基础模型
2. 前端先完成登录和首页骨架
3. 联调习惯创建与今日习惯展示
4. 落打卡与详情页
5. 再进入专注与进度统计
6. 最后补反思系统

## 5. 关键依赖关系

- 登录先于所有鉴权接口
- 习惯模型先于打卡和进度统计
- 打卡记录先于 streak 和完成率计算
- 专注记录先于“专注关联打卡”能力
- 反思能力依赖用户与习惯的基础关系存在

## 6. 每阶段完成定义

每个阶段必须满足以下条件才算完成：

- 数据结构已明确
- 接口契约已明确
- 前后端联调路径已跑通
- 页面具备基础异常处理
- 关键流程有可验证结果

## 7. 里程碑验收清单

### 里程碑 A

- 可以登录
- 可以创建习惯
- 可以看到今日习惯

### 里程碑 B

- 可以打卡
- 可以查看单习惯详情
- 可以看到基础连续天数和完成率

### 里程碑 C

- 可以发起并完成专注
- 可以查看进度总览
- 可以提交与查看反思

## 8. 项目风险与控制

- 微信登录接入复杂度可能影响首阶段节奏
- streak 统计规则如果不统一，会导致前后端展示不一致
- 如果前期把内容和监督一起做，会拖慢核心链路交付
- uniapp 多端差异可能影响页面行为，需要优先按微信小程序校准

## 9. 当前执行建议

如果进入开发阶段，建议先做两件事：

1. 新建 `autom-back-end`，初始化 Spring Boot 工程
2. 按 `docs/api.md` 先实现认证、习惯、打卡相关接口
